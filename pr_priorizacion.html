Priorización basada en valor

Scrum trabaja bajo la priorización, es decir, se abarca en orden primero las tareas definidas por el propietario del producto como más urgentes y prioritarias y después las demás, teniendo en cuenta que las tareas prioritarias sean aquellas que otorgan mayor valor empresarial.
La priorización la realiza el propietario del producto, indicando el orden de las historias de usuario dentro de la lista priorizada de pendientes la cual contiene la lista de requisitos necesarios para realizar el proyecto y encaminarlo al cumplimiento de sus objetivos, el propietario es el encargado del levantamiento de requisitos con el cliente y así comprender exactamente que necesita éste y poder darle el máximo valor empresarial.
Una vez se tengan claros y priorizados los requisitos del cliente, el propietario del producto trabaja en conjunto con el equipo Scrum para identificar los posibles riesgos a los que se pueden enfrentar durante el desarrollo del proyecto y tener planes de mitigación o prevención frente a los riesgos. Un aspecto muy importante para darle valor empresarial al producto y para identificar sus riesgos es poner especial cuidado a las dependencias entre las diferentes variables del proyecto y cómo se comportan entre sí, ya sea basadas en estimación subjetiva (por medio de la experiencia y la observación) o en análisis de mercado ( con herramientas tecnológicas propias para esta tarea).
Se puede concluir que la priorización busca dar entregables de calidad que den valor de negocio en el menor tiempo posible, cumpliendo con los requisitos que suplan las necesidades del cliente por las cuales surgió el proyecto.
